# Judge Script

## Opening

SimGuard protects mobile money agents from processing fraudulent withdrawals after a SIM swap or device takeover.

## 90-second walkthrough

1. Show the guided scenarios at the top of the app.
2. Run the 3-step showcase.
3. For the blocked case, point out:
   - recent SIM swap
   - high-value transfer
   - clear operational action
4. For the flagged case, point out:
   - device change
   - medium-risk review path
5. For the approved case, point out:
   - clean trust signals
   - faster safe processing

## What to emphasize against judging criteria

- Innovation:
  telecom trust signals plus AI decisioning in one agent workflow
- Impact:
  directly addresses mobile money fraud in Sub-Saharan Africa
- Technical feasibility:
  uses multiple CAMARA APIs with a clean fallback strategy
- Scalability:
  can serve agents, fraud desks, and wallet platforms
- Presentation:
  deterministic showcase with live feed and alert updates

## Mentor questions to ask

- Which buyer do you find strongest first: operator fraud team, PSP, or bank wallet team?
- What proof point would make the commercial story stronger in the prototype pitch?
- Which API combination feels most persuasive to judges and future partners?
